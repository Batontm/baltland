# Кластеризация на карте в Baltland

## Кратко
В проекте кластеризация реализована в Leaflet-карте через `react-leaflet-cluster` (обёртка над `Leaflet.markercluster`).
Основной рабочий файл: `components/map/leaflet-catalog-map.tsx`.

---

## Где это используется

- Публичный каталог: `components/map/catalog-interactive-map.tsx` → `LeafletCatalogMap`
- Карта в админке: `components/admin/dashboard/map-tab.tsx` → `LeafletCatalogMap`
- Диалог карты участка: `components/map/plot-map-dialog.tsx` → `LeafletCatalogMap`

Также есть отдельная реализация `components/map/dgis-map.tsx` (прямой Leaflet + плагин с CDN), но в текущем коде она не подключена импортами.

---

## Библиотеки

- `react-leaflet`
- `leaflet`
- `react-leaflet-cluster`

Кластеризатор подключён как:

- `import MarkerClusterGroup from "react-leaflet-cluster"`

---

## Логика по уровням зума

В `leaflet-catalog-map.tsx` задана трёхуровневая модель отображения:

- `CLUSTER_MAX: 12` — кластеры
- `MARKER_MAX: 15` — отдельные маркеры
- `POLYGON_MIN: 16` — полигоны участков

Фактически кластеризация отключается на zoom 13+ через:

- `disableClusteringAtZoom={ZOOM_THRESHOLDS.CLUSTER_MAX + 1}`

И маркерный слой скрывается при переходе к полигонам (плавный fade).

## Про «синаптическое ядро» кластеризации (что именно является ядром логики)

В текущем репозитории нет явного модуля/класса с названием `synaptic core` или `синаптическое ядро`.
Но если под «ядром» иметь в виду центральный алгоритм группировки, то он реализован в двух местах:

1. **Основное рабочее ядро (production-путь)** — `LeafletCatalogMap`:
   - используется готовый движок `react-leaflet-cluster` / `Leaflet.markercluster`;
   - ваше прикладное ядро здесь — это пороги зума + параметры `MarkerClusterGroup` + правила переключения слоёв.

2. **Алгоритмическое ядро микрокластеризации (альтернативная реализация)** — `DGisMap`:
   - поверх обычной кластеризации есть ручная группировка близких маркеров в детальном режиме;
   - принцип: точки объединяются, если разница координат меньше порога;
   - порог адаптивный:
     - `0.001` на среднем зуме,
     - `0.0001` на детальном (`zoom >= detailZoomThreshold`),
   - при `zoom >= 17` микрокластеризация почти отключается (показываются почти все маркеры отдельно).

То есть «ядро» в практическом смысле — это сочетание:
- движка markercluster,
- правил переключения по zoom,
- ручной пост-обработки перекрытий (микрокластеризация в `DGisMap`).

## Как настроен MarkerClusterGroup

Текущие ключевые опции:

- `chunkedLoading` — порционная отрисовка (без фризов при больших объёмах)
- `spiderfyOnMaxZoom={false}` — клик по кластеру зумит, а не «раскладывает пауком»
- `showCoverageOnHover={false}` — без подсветки охвата кластера
- `maxClusterRadius={50}` — радиус агрегации
- `disableClusteringAtZoom={13}` — переход к одиночным маркерам
- `animate={true}`
- `animateAddingMarkers={false}` — меньше лишних анимаций/нагрузки
- `zoomToBoundsOnClick={true}`
- `removeOutsideVisibleBounds={true}` — удаление невидимых маркеров за пределами экрана
- `iconCreateFunction={createClusterIcon}` — кастомный вид кластера

## Визуал кластеров

Кластеры отрисовываются кастомной SVG-иконкой «пин-капля» с числом внутри:

- генератор: `createClusterTeardropSvg(...)`
- иконка кластера: `createClusterIcon(...)`

Сейчас цвет кластера зафиксирован в коде (`#22c55e`, зелёный).

Важно: в типах и UI настроек есть `cluster_marker_color`, но в `createClusterIcon` он пока не используется (цвет берётся константой).

## Производительность и UX-решения

В текущей реализации сделаны важные оптимизации:

1. Кластеры только на низком/среднем зуме, затем переключение на маркеры и полигоны.
2. `removeOutsideVisibleBounds` для разгрузки DOM.
3. Отключён spiderfy (`spiderfyOnMaxZoom={false}`).
4. `animateAddingMarkers={false}`.
5. Полигоны рендерятся только на детальном зуме.
6. Viewport culling для полигонов (`mapBounds.pad(0.3)`) — рисуются только объекты рядом с экраном.
7. На touch-устройствах убраны hover-обработчики.

Итог: карта не перегружается тысячами SVG-элементов на малом зуме и остаётся отзывчивой.

---

## Поток данных (упрощённо)

1. В карту приходят `plots`.
2. В `parsedPlots` нормализуются координаты (центр + полигоны).
3. В зависимости от zoom:
   - low zoom: кластерный слой
   - middle zoom: одиночные маркеры
   - high zoom: полигоны + подписи кадастра
4. Клик по кластеру зумит до bounds, клик по маркеру/полигону выбирает участок.

## Альтернативная реализация (DGisMap)

`components/map/dgis-map.tsx` использует `L.markerClusterGroup` напрямую и пороги из `map_settings`:

- `cluster_zoom_threshold` (по умолчанию 14)
- `detail_zoom_threshold` (по умолчанию 16)

Особенность: там есть дополнительная «микрокластеризация» в детальном режиме для перекрывающихся маркеров.

Но так как компонент сейчас не подключён, фактическое поведение сайта определяется `leaflet-catalog-map.tsx`.

---

## Быстрый чек-лист, если нужно менять кластеризацию

1. Порог кластеров: `ZOOM_THRESHOLDS.CLUSTER_MAX`
2. Радиус: `maxClusterRadius`
3. Поведение по клику: `zoomToBoundsOnClick`, `spiderfyOnMaxZoom`
4. Производительность: `chunkedLoading`, `removeOutsideVisibleBounds`, `animateAddingMarkers`
5. Внешний вид: `createClusterIcon` + CSS `.custom-cluster-icon`

## Вывод

Текущая кластеризация сделана грамотно для большого числа участков:

- агрегация на низком зуме,
- плавный переход на маркеры,
- детальная геометрия только при приближении,
- набор оптимизаций против лагов.

Это и обеспечивает «быстрый обзор на отдалении + точность при приближении».
