# Деплой на продакшен (85.198.85.239)

> [!IMPORTANT]
> **SSH доступ**: Используется SSH-ключ (`root`). Пароль не требуется.
> **Переменные окружения (.env)**: На сервере уже настроены правильные `.env` и `.env.docker`. **НЕ деплойте локальные .env файлы**, так как они отличаются от серверных.
> **База данных**: Все изменения касаются только приложения (контейнер `rkkland_web`). База данных Supabase (`volumes`) защищена и не затрагивается деплоем.

## 🚀 Команда для автоматического деплоя

Просто запусти эту команду локально для обновления кода на сервере:

```bash
bash scripts/deploy.sh
```

IP сервера (`85.198.85.239`) и путь (`/var/www/rkkland`) захардкожены в скрипте. Поддерживаемый флаг:
- `--with-seed` — дополнительно выгрузить локальные данные Supabase и импортировать их на VPS.

---

## 🔍 Что делает этот процесс:

1. **Синхронизация**: `rsync` копирует файлы кода, исключая `node_modules`, `.next`, `.git` и все `.env` файлы.
2. **Безопасность**: База данных Supabase (`infra/supabase/docker/volumes`) **не синхронизируется**, чтобы не затереть данные.
3. **Сборка**: Контейнер `rkkland_web` пересобирается прямо на сервере, используя локальные для сервера `.env` файлы.
4. **Запуск**: Приложение перезапускается в фоновом режиме.

## 🛠 Полезные команды на сервере

Если нужно проверить состояние или логи:

```bash
# Перейти в папку проекта
ssh root@85.198.85.239 "cd /var/www/rkkland && bash"

# Посмотреть статус контейнеров
ssh root@85.198.85.239 "docker ps | grep rkkland"

# Посмотреть свежие логи приложения
ssh root@85.198.85.239 "docker logs -f --tail 100 rkkland_web"
```

## ⚠️ Решение проблем

Если контейнер завис или не обновляется:
1. Удалить старый контейнер: `ssh root@85.198.85.239 "docker rm -f rkkland_web"`
2. Запустить деплой заново.
