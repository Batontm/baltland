# Plan: Бот для MAX — поиск земельных участков Baltland

> Дата составления: март 2026  
> Контекст: существующий проект Next.js + Supabase на baltland.ru

---

## 1. Что это такое и зачем

Бот в мессенджере **MAX** (max.ru, бывший VK Мессенджер), который позволяет пользователю:

1. Выбрать фильтры (район, цена, площадь, тип земли, коммуникации, рассрочка)
2. Получить список подходящих участков прямо в чате
3. Посмотреть параметры каждого участка
4. Оставить заявку на просмотр — не выходя из мессенджера

**Почему MAX:**
- Аудитория MAXа пересекается с целевой (Россия, недвижимость)
- Официальная TypeScript-библиотека от команды MAX
- Можно работать на Webhook — на нашем VPS уже есть HTTPS
- Telegram-бот у нас уже есть → MAX-бот будет вторым каналом

---

## 2. Ограничения платформы (важно прочитать первым)

С **августа 2025** MAX изменил правила для ботов:

| Требование | Детали |
|---|---|
| **Только юрлица РФ** | ИП, самозанятые, физлица — не допускаются |
| **Верификация организации** | Нужен ИНН, срок ~48 рабочих часов |
| **Модерация бота** | Заполнить карточку бота → модерация → токен |
| **Обязательно HTTPS** | Webhook работает только по HTTPS (у нас уже есть) |
| **Rate limit** | 30 rps к `platform-api.max.ru` |

**Чтобы начать:**
1. Войти в MAX для бизнеса: https://dev.max.ru/docs/maxbusiness/connection
2. Зарегистрировать юрлицо (ИНН компании)
3. Создать профиль организации
4. Написать `@MasterBot` в MAXе для создания бота → получить токен
5. Заполнить карточку бота → отправить на модерацию
6. Или написать на `partner@max.ru` с просьбой о доступе

## 3. Техническая архитектура

### 3.1 Технологический стек

```
MAX мессенджер
    ↓ Webhook HTTPS POST
[Новый микросервис: /api/max-bot/webhook]  ← в нашем Next.js на VPS
    ↓
Supabase (наша БД land_plots)
    ↓ JSON-ответ
MAX мессенджер ← отправляет ответ пользователю
```

### 3.2 Варианты реализации

**Вариант A (рекомендуется): Встроить в существующий Next.js**
- Новый route: `app/api/max-bot/webhook/route.ts`
- Библиотека: `@maxhub/max-bot-api` (официальная TypeScript)
- Подключается к существующему Supabase (те же env-переменные)
- Деплой: в рамках `bash scripts/deploy.sh`
- Плюс: всё в одном репозитории, не нужен отдельный сервер

**Вариант B: Отдельный Node.js сервис**
- Отдельный процесс на VPS (pm2)
- Плюс: изолирован от основного приложения
- Минус: дополнительная сложность деплоя

**Выбор: Вариант A** — проще, уже есть инфраструктура.

### 3.3 Новая env-переменная

```env
MAX_BOT_TOKEN=your_token_here
```

Добавить в:
- `.env.local` (dev)
- `/var/www/rkkland/.env.docker` (prod, как для Telegram)

## 7. Ключевые технические решения

### 7.1 Состояние диалога (session)

Бот должен помнить, какие фильтры выбрал пользователь.

**Варианты хранения:**
- **In-memory Map** `userId → { filters, step, page }` — просто, достаточно для MVP
- **Redis** — для масштабирования (VPS уже есть, можно добавить контейнер)
- **Supabase** таблица `bot_sessions` — надёжно, но избыточно

**Рекомендация для MVP:** In-memory Map. При перезапуске состояние сбрасывается — некритично.

### 7.2 Запрос к Supabase

Переиспользуем существующий `lib/supabase/admin.ts`:

```typescript
const { data } = await supabaseAdmin
  .from("land_plots")
  .select("id, title, location, district, area_sotok, price, land_status, ...")
  .eq("is_active", true)
  .eq("district", filters.district)
  .lte("price", filters.maxPrice)
  .limit(5)
  .range(page * 5, (page + 1) * 5 - 1)
```

### 7.3 Пагинация в боте

- Показывать по **5 участков** за раз
- Кнопка `Следующие 5 →` передаёт `page` в callback_data
- `callback_data` формат: `search:page:2` (до 64 байт — лимит MAX)

### 7.4 Webhook vs Long Polling

- **Разработка:** Long Polling (`bot.start()`) — не нужен публичный URL
- **Продакшн:** Webhook — `/api/max-bot/webhook` на нашем HTTPS VPS

```typescript
// dev
bot.start()

// prod
await bot.api.subscriptions.subscribe({ url: "https://baltland.ru/api/max-bot/webhook" })
// затем в route.ts обрабатываем POST-запросы
```

## 8. Создание заявки (lead)

При нажатии «Записаться на просмотр» бот:

1. Просит поделиться контактом (стандартная кнопка MAX)
2. Создаёт запись в таблице `leads` (уже существует):
   ```sql
   INSERT INTO leads (name, phone, source, plot_id, notes)
   VALUES ('Имя из профиля', '+7...', 'max_bot', 'plot_uuid', 'Через бот MAX')
   ```
3. Отправляет уведомление в Telegram (через уже существующий `lib/telegram.ts`)

## 12. Риски и ограничения

- **Регистрация юрлица** — основной блокер. Без верифицированного юрлица бот нельзя вывести в прод (только личный тест).
- **Модерация** — карточка бота может не пройти с первого раза.
- **Аудитория** — MAX меньше Telegram в Калининградской области. Эффективность зависит от продвижения.
- **In-memory state** — при перезапуске Docker-контейнера состояние диалогов сбрасывается. Для MVP некритично.
